Uhambo v63 검증 가이드

결함 수정 (linked 문서 정리, 삭제 순서, 구독 에러 처리) — 실제 앱에서 아래 순서대로 입력/조작하며 확인

1. Booking 삭제 시 연결된 Expense/Itinerary 함께 삭제 예약 ↔ 연결항목

입력 (Booking → Accommodation 추가)
필드값
Name테스트 호텔
Cost100000
CurrencyKRW
Check-in2026-07-01, 15:00
Check-out2026-07-03, 11:00
StatusBooked
저장.
예상 결과 1 — 저장 직후:
  • Expenses 탭에 "테스트 호텔" 100,000 KRW 항목 자동 생성
  • Itinerary 탭 2026-07-01에 "Check-in: 테스트 호텔", 2026-07-03에 "Check-out: 테스트 호텔" 항목 자동 생성
조작 — Booking 탭에서 "테스트 호텔" 항목 열기 → Delete → 확인
예상 결과 2 — 삭제 후:
  • Booking 목록에서 "테스트 호텔" 사라짐
  • Expenses 탭의 "테스트 호텔" 항목도 함께 사라짐
  • Itinerary 탭의 Check-in/Check-out 2건도 함께 사라짐
v62 이전: 위 항목들이 모두 정상 삭제되긴 했으나, 내부적으로 컬렉션 전체를 스캔하는 비효율적 방식이었음 — 동작 자체는 동일하게 보여야 함 (회귀 없음 확인용)

2. Activity 비용 입력 시 Expense 자동 생성 / 비용 삭제 시 Expense 자동 제거 신규 동작

입력 (Activities → 추가)
필드값
Name박물관 관람
Date2026-07-02
Cost15000
CurrencyKRW
저장.
예상 결과 1 — Expenses 탭에 "박물관 관람" 15,000 KRW 항목이 (category: activity) 자동 생성됨
조작 — "박물관 관람" Activity를 다시 열어 Cost 필드를 비움 → 저장
예상 결과 2 (이번 수정의 핵심) — Expenses 탭에서 "박물관 관람" 항목이 자동으로 사라짐
v62 이전: Cost를 비워도 Expenses에 0원짜리 "박물관 관람" 항목이 그대로 남아있었음 (고아 데이터)
조작 — 다시 Cost에 15000 입력 후 저장
예상 결과 3 — Expenses 탭에 "박물관 관람" 15,000 KRW 항목이 다시 생성됨

3. Activity 삭제 시 연결된 Expense/Itinerary 함께 삭제 활동 ↔ 연결항목

입력 — 위 "박물관 관람" 항목 (Cost=15000, Date=2026-07-02, Time=14:00 입력)을 그대로 사용
사전 확인 — Itinerary 탭 2026-07-02에 "박물관 관람" 항목이 보여야 함, Expenses 탭에도 15,000 KRW 항목 존재
조작 — Activities 탭에서 "박물관 관람" 열기 → Delete → 확인
예상 결과
  • Activities 목록에서 사라짐
  • Expenses 탭의 15,000 KRW 항목도 함께 사라짐
  • Itinerary 탭의 "박물관 관람" 항목도 함께 사라짐

4. 크루즈 다중 Port call 저장 성능 선택 — 개발자 도구 필요

입력 (Booking → Cruise 추가)
필드값
Ship Name테스트 크루즈
Embark2026-08-01
Disembark2026-08-05
Port Call 1제주, 2026-08-02, 도착 08:00
Port Call 2부산, 2026-08-03, 도착 09:00
Port Call 3오사카, 2026-08-04, 도착 10:00
조작 — 저장 전, 브라우저 개발자 도구(F12) → Network 탭 → "Fetch/XHR" 또는 "WS" 필터로 Firestore 요청을 관찰하며 저장 버튼 클릭
예상 결과 — 저장 시 Firestore 채널로 가는 요청/쓰기 수가 적음 (port call 3개 기준 풀스캔이 발생하지 않음). Itinerary 탭에 제주/부산/오사카 3개 입항 일정이 각 날짜에 정상 표시됨.
정확한 요청 수 비교는 v62 빌드와 나란히 띄워 비교해야 의미 있음 — 일반 사용자는 "정상적으로 3개 입항 일정이 생성되는지"만 확인하면 충분

5. 네트워크 끊김 시 빈 화면 대신 에러 메시지 표시 선택 — 오프라인 토글 필요

조작 — 앱이 정상 로드된 상태에서 개발자 도구 → Network 탭 → "Offline" 체크 → Booking/Itinerary/Activities/Expenses 탭을 차례로 새로고침 또는 전환
예상 결과 — Firestore 캐시된 데이터가 없는 신규 트립의 경우, 목록 영역에 ⚠️ 아이콘과 에러 메시지가 표시됨 (완전히 빈 화면이 아님). 단, Firestore 오프라인 캐시(persistentLocalCache)가 활성화되어 있어 이전에 본 데이터가 있으면 캐시된 데이터가 그대로 보일 수 있음 — 이 경우 정상 동작.
