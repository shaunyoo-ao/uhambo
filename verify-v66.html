Expenses/Activities/Booking 검색 + Expenses 정렬 — 실제 앱에서 아래 순서대로 입력/조작하며 확인

1. Expenses 검색 + 정렬 신규 동작

사전 준비 — Expenses에 아래 3건을 추가:
TitleAmountCurrencyDateCategory
도쿄 스시30000JPY2026-07-01food
지하철 패스3000JPY2026-07-02transport
호텔 비용200USD2026-07-03accom
조작 1 — 검색 — 검색창에 스시 입력
예상 결과 — 목록에 "도쿄 스시"만 남고 나머지 2건은 사라짐. 상단 요약 카드(총액/카테고리)는 그대로 유지됨 (필터링되지 않음)
조작 2 — 검색 결과 없음 — 검색창에 없는항목 입력
예상 결과 — "검색 결과 없음" (또는 EN: "No matches") 빈 상태 메시지 표시
조작 3 — 정렬 (금액) — 검색창을 비우고, 정렬 셀렉트를 Amount로 변경
예상 결과 — 통화가 다른 항목들이 표시 통화(예: KRW)로 환산된 금액 기준 내림차순으로 정렬됨 (예: 호텔 비용(USD 200 ≈ 더 큰 금액)이 최상단에 오는 등 — 실제 환율에 따라 순서 확인)
조작 4 — chip 전환에도 검색 유지 — 검색창에 도쿄 입력 후, 카테고리 칩을 "food" → "all"로 전환
예상 결과 — 칩을 전환해도 검색어 "도쿄"가 그대로 남아있고, 검색 필터가 계속 적용됨
조작 5 — 트립 전환 시 검색 리셋 — 검색어를 입력한 상태에서 다른 트립으로 전환했다가 다시 Expenses로 돌아옴
예상 결과 — 검색창과 정렬, 카테고리 필터가 모두 초기 상태("all" / "date" / 빈 검색)로 리셋됨

2. Activities 검색 신규 동작

사전 준비 — Activities에 "박물관 관람"(2026-07-02), "해변 산책"(2026-07-03) 2건 추가
조작 1 — 검색창에 박물관 입력
예상 결과 — "박물관 관람"이 속한 2026-07-02 날짜 그룹만 표시되고, "해변 산책"이 속한 2026-07-03 날짜 그룹 헤더는 완전히 사라짐 (빈 헤더가 남지 않음)
조작 2 — 검색창을 비움
예상 결과 — 두 날짜 그룹 모두 다시 표시됨

3. Booking 검색 신규 동작

사전 준비 — Booking에 Accommodation "테스트 호텔"(name), Travel(airline="Korean Air"), Rent(rentalCompany="Hertz") 각 1건씩 추가
조작 1 — 검색창에 Hertz 입력
예상 결과 — Rent 카드만 표시됨 (Accommodation/Travel 카드는 사라짐)
조작 2 — 검색창에 Korean 입력
예상 결과 — Travel 카드(airline="Korean Air")만 표시됨
조작 3 — 게스트 모드 — 게스트 코드로 로그인하여 Booking 탭 확인
예상 결과 — 게스트 모드에서도 검색창이 보이고 정상 동작함 (읽기 전용 UI이므로 추가 가드 불필요)
