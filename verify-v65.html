Uhambo v65 검증 가이드

모달 ESC 닫기, 오프라인 배너, 스켈레톤 로딩, 터치 타겟 확대 — 실제 앱에서 아래 순서대로 조작하며 확인

1. 모달 ESC 키로 닫기 신규 동작

조작 1 — 아무 화면(예: Activities)에서 + 버튼을 눌러 추가 모달을 연 뒤, 키보드 Esc 키를 누름
예상 결과 — 모달이 닫힘 (저장되지 않은 입력값은 사라짐, 정상 동작)
조작 2 — 우선순위 확인 (Confirm)
  1. 아무 항목이나 열어 모달의 Delete 버튼을 눌러 "삭제 확인" 다이얼로그를 띄움
  2. 이 상태에서 Esc 키를 누름
예상 결과 — 확인 다이얼로그가 열려있는 동안에는 Esc가 무시됨 (다이얼로그도 모달도 닫히지 않음). "확인 다이얼로그를 닫으려면 Cancel/OK 버튼을 눌러야 함" — 의도된 동작
조작 3 — 우선순위 확인 (계산기)
  1. Expense/Activity 추가 모달에서 비용 입력란 옆 계산기(🧮) 버튼을 눌러 계산기 오버레이를 엶
  2. 이 상태에서 Esc 키를 누름
예상 결과 — 계산기가 열려있는 동안에는 Esc가 무시됨 (계산기도 뒤의 모달도 닫히지 않음)

2. 오프라인 배너 신규 동작

조작
  1. 앱이 정상 로드된 상태에서 개발자 도구(F12) → Network 탭 → "Offline" 체크
  2. 화면 상단 헤더 바로 아래를 확인
예상 결과 1 — 헤더 아래에 노란색(amber) 띠 배너가 나타나고, 다음 텍스트가 표시됨:
  • EN: "You are offline — changes will sync when reconnected"
  • KO (설정에서 한국어로 전환 시): "오프라인 상태입니다 — 연결되면 동기화됩니다"
조작 — Network 탭에서 "Offline" 체크 해제 (온라인 복귀)
예상 결과 2 — 배너가 즉시 사라짐

3. 스켈레톤 로딩 화면 Slow 3G 권장

조작
  1. 개발자 도구 → Network 탭 → 속도 제한을 "Slow 3G"로 설정
  2. Expenses, Activities, Booking, Itinerary 탭을 차례로 전환
예상 결과 — 데이터 로딩 중에는 회전 스피너(circle spinner) 대신, 카드 모양의 회색 음영(shimmer) 블록 4개가 표시됨. 데이터가 도착하면 shimmer가 실제 목록으로 교체됨
v64 이전: 화면 중앙에 작은 회전 스피너만 표시되었음

4. 터치 타겟 확대 모바일 에뮬레이션 권장

조작
  1. 개발자 도구 → 모바일 기기 에뮬레이션 모드 (예: iPhone SE)
  2. 헤더의 새로고침/설정/패킹 아이콘 버튼을 손가락 크기로 탭
  3. Activities 또는 Packing 탭에서 체크박스를 항목 가장자리 근처(테두리 살짝 바깥)를 탭
  4. Expense/Activity 모달에서 링크 추가 후 ✕ 삭제 버튼을 탭
  5. chip(필터 카테고리) 버튼들을 연속으로 탭
예상 결과
  • 헤더 아이콘 버튼이 이전보다 더 커진 정사각형(40×40px)으로 보이고 탭이 잘 인식됨
  • 체크박스는 시각적 크기는 동일하지만, 테두리 바로 바깥(약 10px)을 탭해도 토글됨
  • 링크 삭제(✕), 모달 닫기(X), packing 체크/삭제 버튼이 이전보다 여유 있게 탭됨 (오탭 감소)
  • chip 버튼의 세로 폭이 약간 넓어져 연속 탭이 더 편해짐
