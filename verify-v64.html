Uhambo v64 검증 가이드

XSS escape 적용, packing 재구독 버그 수정, SW precache 보강 — 실제 앱에서 아래 순서대로 입력/조작하며 확인

1. XSS 스크립트 입력 시 리터럴 텍스트로만 표시 전 화면 공통

입력값 (모든 케이스 공통)
필드 종류입력값
제목/이름류<img src=x onerror=alert(1)>
메모/설명류<script>alert('xss')</script>
조작 1 — Expenses → 추가 → Title에 위 <img...> 입력, Amount=1000, 저장
조작 2 — Activities → 추가 → Name에 위 <img...>, Notes에 위 <script> 입력, 저장
조작 3 — Booking → Accommodation → 추가 → Name에 위 <img...>, Notes에 위 <script> 입력, 저장
조작 4 — Itinerary → 추가 → Title/Location/Notes에 위 값들 입력, 저장
조작 5 — Packing → 항목 추가 → 제목에 위 <img...> 입력
예상 결과 — 모든 화면(목록 카드, 모달 재오픈 시 입력값, Itinerary 캘린더 팝업, 지도 핀 팝업 포함)에서:
  • <img src=x onerror=alert(1)> 텍스트가 그대로 글자로 보임 (이미지 깨짐 아이콘이나 alert 팝업 없음)
  • <script>alert('xss')</script> 텍스트도 그대로 글자로 보임, alert 없음
v63 이전: alert 팝업이 뜨거나 이미지 깨짐 아이콘이 표시되었음 (XSS 취약점)
조작 6 — 이중 escape 확인 — 위에서 저장한 항목들을 다시 열어 수정 모달을 띄운 뒤, 아무것도 바꾸지 않고 다시 저장 (총 2회 편집-저장)
예상 결과 — 2회 편집 후에도 화면에 &lt;, &amp;lt; 같은 이스케이프 문자 누적(드리프트)이 없어야 함 — 처음과 동일하게 <img src=x onerror=alert(1)> 형태 그대로 표시

2. Packing 재구독/FAB 중복 버그 수정 신규 동작

조작
  1. 하단 탭에서 Packing으로 이동
  2. 다른 탭(예: Dashboard)으로 이동했다가 다시 Packing으로 3번 이상 왕복 이동
  3. 개발자 도구 → Elements 탭에서 .fab 클래스 버튼 개수 확인
예상 결과 1 — 화면 우측 하단에 + FAB 버튼이 1개만 보임 (DOM에 .fab 요소가 1개만 존재)
v63 이전: Packing 탭을 재방문할 때마다 FAB가 중복 추가되어 여러 개 겹쳐 보였음
조작 — Packing 항목 하나의 체크박스를 1번만 클릭해 packed 상태로 토글. 개발자 도구 → Network 탭에서 Firestore 쓰기 요청 수 확인
예상 결과 2 — 체크박스 1번 클릭 시 Firestore 업데이트가 1회만 발생함
v63 이전: 탭을 여러 번 재방문한 뒤에는 이전 구독이 살아있어 동일 클릭에도 중복 업데이트가 발생할 수 있었음

3. Service Worker 캐시 — packing/calculator 오프라인 동작 개발자 도구 필요

조작 1 — 캐시 확인
  1. 앱을 새로고침하여 새 Service Worker(v64)가 활성화되도록 함 (Settings 또는 로그인 화면 하단 버전이 1.2.64인지 확인)
  2. 개발자 도구 → Application 탭 → Cache Storage → app-shell-v64 펼치기
예상 결과 — app-shell-v64 캐시 목록에 다음 항목들이 모두 존재:
  • /js/pages/packing.js
  • /js/calculator.js
  • /icons/icon-maskable-192.png
  • /icons/icon-maskable-512.png
v63 이전: 이 파일들이 precache 목록에 없어 오프라인 시 로드 실패 가능성 있었음
조작 2 — 오프라인 동작 — 개발자 도구 → Network 탭 → "Offline" 체크 → 앱 새로고침 → Packing 탭 이동, 비용 계산기(🧮) 버튼 클릭
예상 결과 — 오프라인 상태에서도 Packing 페이지가 정상 렌더링되고, 계산기 모달이 정상적으로 열림 (빈 화면이나 로드 에러 없음)
버전: sw.js v64 / app.js APP_VERSION 1.2.64 — Settings 팝업 또는 로그인 화면 하단에서 버전 확인 가능
